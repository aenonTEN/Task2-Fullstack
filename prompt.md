"Develop an integrated management platform for pharmaceutical compliance and talent operations, designed for business specialists, compliance administrators, recruitment specialists, and system administrators. The frontend is built with Vue.js and features an English interface. After login, different menus and data scopes (by institution/department/team) are displayed based on RBAC. Key operations include clear feedback and secondary confirmation. 

On the recruitment side, support bulk resume import and structured field editing (name, contact information, skills, experience, etc.), with configurable tags and custom fields. Duplicate records triggered by the same phone number or ID number are merged. Intelligent search provides filtering and sorting by keywords/skills/education/experience/time, outputting a 0–100 match score with explainable reasons (e.g., ""3 skills matched, experience requirements met"") and recommending similar candidates/similar positions. 

On the compliance side, maintain client/supplier qualification profiles, support expiration reminders (default: highlighted in red 30 days in advance) and automatic deactivation upon expiration. Set purchase restrictions for controlled/prescription medications (requiring prescription attachments, limiting purchases to once per 7 days per client) and retain records. 

For case handling, implement a unique numbering rule (e.g., YYYYMMDD-institution-6-digit serial), mandatory key fields, and duplicate submission blocking within 5 minutes. Support assignment, processing records, attachment archiving, and status transitions to form a searchable ledger. 

The backend is built with Go and Gin to implement local RESTful APIs, with all functionality running in an offline intranet deployment. MySQL stores core tables (users and roles, data scopes, candidates and positions, qualifications and restrictions, case ledgers, attachment indexes, audit logs). Authentication uses local username + password validation, with passwords requiring at least 8 characters and encrypted using bcrypt. Session tokens have a default validity of 8 hours and support logout invalidation. APIs undergo secondary validation by role and data scope. Sensitive fields (ID numbers, phone numbers) are encrypted in storage and desensitized in list displays. 

Import/export and attachments are handled via the local file system, with uploads validated against a format whitelist and deduplicated using SHA256 fingerprints. Support resumable chunked uploads and offline temporary storage. 

Audit logs record permission changes and modifications to fields related to resumes/qualifications/cases/fees, capturing the operator, time, request source, and field differences (before/after values). Logs are searchable by conditions and exportable, with an append-only, non-modifiable policy to ensure non-repudiation."
